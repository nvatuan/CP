D. Cycle Free Flow
time limit per test
4 seconds
memory limit per test
256 megabytes
input
standard input
output
standard output

You are given an undirected connected weighted graph of n nodes which contains no cycles. For q different pairs of nodes, we would like you to find the maximum flow from the first node in the pair to the second node in the pair. See the definition for max flow below if you don't know what it means.

Also, please answer these queries online. Originally, I was going to force you to do this by encrypting stuff, but that would make this problem harder to read and more annoying. So, on your honor, please answer the first query before reading the second query, et cetera. After all, this is just practice for you to get better, right?

The maximum flow in a graph between two nodes aa and bb can be defined as follows: You may pick any path from aa to bb such that every edge on that path has a positive weight. Then you may subtract 1 from the weight of every edge on that path, and add 1 to your answer. Keep doing this as many times as you would like. The "maximum flow" between the two nodes is the greatest that your answer could possibly be, if you choose your paths optimally at every step.

Input

The first line will contain a two integers: nn and mm, the number of nodes and the number of edges in the graph. The next mm lines will each contain three integers: the endpoints uu and vv of an undirected edge, and the initial weight of that edge ww.

The next line will contain a single integer qq: the number of queries you must answer.

Each of the next q lines will contain two integers: aa and bb, the two nodes you should find the max flow between.

2≤n,m,≤3∗1052≤n,m,≤3∗105

1≤a,b,≤n1≤a,b,≤n a≠ba≠b

1≤w≤1091≤w≤109

u≠vu≠v

Additional constraint on input: the given graph is connected and has no cycles, self-loops, or multi-edges.

Output

For each query, please print a single integer: the max flow between nodes aa and nodes bb for that query.

E. Filthy Rich Trees

4 seconds
memory limit per test
256 megabytes
input

output

You may have heard the saying "money doesn't grow on trees." Obviously, whoever came up with it stunk at both botany and programming. You've learned that creating trees which grow obscene amounts of money is so trivial, that it's left as an exercise to the reader. Time for the real challenge: profiting off your invention!

You have rooted your money-tree at node 11, and each node has a specified moola value. The amount of money that some subtree rooted at node xx will produce is equal to the product of the moola values of all nodes in xx's subtree. Initially, all nodes will have a moola value of 1.

Now, qq times, one of two things happen, given to you in the form of "t x y":

If t=1t=1, you give node xx a special fertilizer which sets its moola value to yy.

If t=2t=2, a customer comes in and asks "how many times more valuable is subtree xx than subtree yy"? Formally, what is the value of subtree xx divided by the value of subtree yy. Also, you don't like printing massive numbers, so subtree xx is more than 109109 times greater than subtree yy, only print "1000000000".

Input

The first line will contain a single integer nn, the number of nodes in the tree. n−1n−1 lines follow, each containing two different integers, describing the edges of the tree. Additional constraint on input: these edges will form a tree.

After that there will be a single integer qq: the number of queries. qq line follow, each of the form txytxy, as described above.

1≤n,q≤3∗1051≤n,q≤3∗105

1≤t≤21≤t≤2

1≤x,y≤n1≤x,y≤n

Output

For each query of type 2, print a single line with one real number: how many times more valuable subtree xx is than subtree yy. HOWEVER, if this value is >= 109109, then print just 10000000001000000000 instead.

Your answer will be accepted if your answers to all queries have absolute or relative error of at most 10−610−6.
